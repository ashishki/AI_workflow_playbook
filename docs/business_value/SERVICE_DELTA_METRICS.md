# Service Delta Metrics

## Purpose

AI value is not only FTE savings. Many useful workflows improve cycle time,
SLA, error rate, throughput, or service differentiation.

## Metric Types

| Metric | Example |
|--------|---------|
| Cycle time delta | Request-to-resolution time decreases |
| SLA delta | More cases handled inside target |
| Error rate delta | Fewer wrong claims, bad fields, or missed cases |
| Throughput delta | More items processed per day with same team |
| Quality delta | Better citation precision or lower correction rate |
| Operator load delta | Less repetitive review time |
| Service delta | New response speed or coverage previously impossible |
| Risk delta | Lower audit gap, fewer unauthorized actions |

## Rule

Every value claim needs a metric, baseline, measurement method, and forbidden
claim until evidence exists.
