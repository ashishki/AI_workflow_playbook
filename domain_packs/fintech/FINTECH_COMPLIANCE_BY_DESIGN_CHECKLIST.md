# Fintech Compliance-by-Design Checklist

## Checklist

| Check | Status |
|-------|--------|
| Named regulatory/compliance obligations identified | pending |
| Data classification for PII, financial, account, transaction data | pending |
| KYA record for each agent | pending |
| Tool permission classes and blocked actions documented | pending |
| Human approval for high-risk actions | pending |
| Audit log schema and retention defined | pending |
| Transaction/action limits defined | pending |
| Judge calibration status documented if judge is used | pending |
| Data readiness and eval readiness complete | pending |
| Incident/disable process documented | pending |

## Rule

Compliance evidence is a launch gate when the use case touches payments,
accounts, credit, KYC/AML, customer financial data, or regulated reporting.
