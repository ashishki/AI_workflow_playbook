# Designer Time Control

## Problem

Designers miss timesheets or write vague work logs, making project accounting
late and inaccurate.

## AI Opportunity

Summarize task context and draft timesheet entries. This is not surveillance:
the designer owns approval and edits.

## Data Required

Task tracker items, calendar/project metadata, optional commit or file activity
summaries, and approved historical timesheets.

## Risk and HITL

Risk: privacy overreach, false attribution, manager misuse. Designer approves
entries; manager sees only approved timesheet data.

## Evaluation Plan

False positive rate, manual correction time, timesheet completion rate, and
operator acceptance.

## MVP Scope

Reminder plus draft timesheet suggestion for one project team.

## Production Hardening

Access limits, opt-out rules, audit log, retention policy, and no raw activity
feed exposed to managers.
